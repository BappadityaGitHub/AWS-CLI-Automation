Topics 6, 7 & 8: Debugging, Redirection, and Logic

Topic 6: Script Debugging & Fixes

Ensure your production scripts are bulletproof by avoiding common variable and array traps.

ProblemThe FixIndustry Logic
Overwriting VarsLOG1="app" vs LOG2="sys"Never let a variable delete its predecessor.
Array Indexing${ARR[0]}Use curly braces and index numbers to pluck specific data.
Space Trap"$VARIABLE"Always use double quotes to handle Windows/Linux spaces.
Reserved VarsAvoid $0 for searching$0 is the script name; don't search for it in logs!
Pro-Tip: Use bash -x script.sh to watch your code run line-by-line.

Topic 7: Pipes & Redirection Logic

Mastering the "Plumbing" of Linux Data Flow.

Pipe (|)
The Water Pipe
Pours data from one tool into another.
2>&1
The Merge
Sends errors (2) to the same place as success (1).
/dev/null
The Black Hole
Deletes unwanted noise silently.
# Example: Silent Disk Check
df -hP / | awk 'NR==2 {print $5}' 2>/dev/null

Topic 8: The Exit Status ($?) & Identity

Knowing the difference between "Who I am" and "How I performed."

SymbolMeaningThe Mentor's Trick
$0Script NameThe "Identity" (The face of the script).
$?Exit StatusThe "Grade" (The result of the last task).
0Success"Zero Problems Found" - The path is clear.
1-255Failure"Error ID Number" - Tells you exactly what broke.
# Professional Success Check
if [ $? -eq 0 ]; then
    echo "SUCCESS: Task finished correctly!"
fi
