 Step 2 .The "Naked Server" Setup (One-Time)
Before you can do anything, your EC2 needs the tools and an identity.

## 2. PHASE TWO: Fixing the "HTTPS/Permission Denied" Trap
If you cloned via HTTPS or get "Permission Denied," you must establish a Secret Handshake (SSH).
Step A: Generate the Key on EC2

ssh-keygen -t ed25519 -C "bappadityasahoo23@gmail.com"# Press ENTER for default location# Press ENTER twice for NO passphrase (standard for easy automation)

Step B: Link to GitHub

   1. See your key: cat ~/.ssh/id_ed25519.pub
   2. Copy the output.
   3. Go to GitHub Settings > SSH Keys > New SSH Key. Paste it and save.

Step C: The "Fix-It" Command (Switching from HTTPS to SSH)
If your folder was cloned with HTTPS, run this inside the folder to stop the password prompts:

git remote set-url origin git@github.com:BappadityaGitHub/AWS-CLI-Automation.git

Step D: Test the Connection

ssh -T git@github.com# Type 'yes' if asked. Look for: "Hi BappadityaGitHub! You've successfully authenticated"

## 3. PHASE THREE: The Production Workflow (The Daily Routine)
This ensures your Laptop, EC2, and GitHub never conflict.

| Time | Goal | Command | Why? |
|---|---|---|---|
| ☀️ Start | Health Check | ssh -T git@github.com | Confirms the "Handshake" is still active. |
| ☀️ Start | Sync Down | git pull origin main | Crucial: Gets updates you made on your Laptop. |
| 💻 Work | Stage Changes | git add . | Includes all changes, even new empty files. |
| 💻 Work | Local Save | git commit -m "feat: updated automation" | Creates a snapshot in history. |
| 🌙 End | Cloud Save | git push | Uploads work. No password will be asked! |

------------------------------
## 4. PHASE FOUR: Security with .gitignore
In production, you never upload secret keys, .pem files, or temporary logs.

* Create the file: nano .gitignore
* What to put inside (Example):

# Ignore AWS Secret Keys
.aws/
credentials.txt

# Ignore Private SSH Keys
*.pem
id_ed25519

# Ignore OS temporary files
.DS_Store
thumbs.db

* Result: When you run git add ., Git will see these files but refuse to upload them.

------------------------------
## 5. THE TRIANGULAR SYNC EXPLAINED

* EC2 to GitHub: You push your automation scripts from the cloud.
* GitHub to Laptop: On your laptop, you run git pull to see what you did on EC2.
* Laptop to GitHub: You edit a script on your laptop and git push.
* GitHub to EC2: On EC2, you run git pull to get the polished script.
* Direct Laptop-EC2 connection? Not needed. GitHub is the middleman.

------------------------------
## 🛠️ Career-Level "Emergency" Commands

* "I made a mistake in my code!": git restore <filename> (Reverts file to the last save).
* "I initialized the wrong branch!": git branch -m master main (Renames branch to main).
* "Is my connection secure?": git remote -v (Should show git@github.com, NOT https://).
